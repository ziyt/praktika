# Магазин обуви

Десктоп-приложение для управления каталогом товаров и заказами магазина обуви.
Реализовано на **Python + PyQt6**, данные хранятся в **SQLite**.

## Возможности

- Каталог товаров с карточками (фото, цена, скидка, остаток на складе).
- Поиск по 7 полям (артикул, наименование, описание, категория, производитель, поставщик, единица измерения).
- Сортировка по количеству на складе, фильтр по поставщику.
- Просмотр заказов и их состава.
- Добавление, редактирование и удаление товаров (для администратора).
- Ролевая модель: Гость, Клиент, Менеджер, Администратор.

## Требования

- Python 3.9 или новее
- PyQt6

## Установка

```bash
git clone https://github.com/ziyt/praktika.git
cd praktika
pip install PyQt6
```

## Запуск

```bash
python3 main.py
```

При первом запуске будет использована готовая база `db/shoe_store.db`.
Если её нет — пересоздать можно командой:

```bash
python3 db/seed.py
```

## Структура проекта

```
main.py                  — точка входа
core/
  auth.py                — аутентификация и роли
  db.py                  — подключение к SQLite
  repository.py          — SQL-запросы
  image_utils.py         — работа с фото товаров
  paths.py               — пути к ресурсам
  styles.py              — общая QSS-таблица стилей
ui/
  login_window.py        — окно входа
  products_window.py     — каталог товаров
  product_edit_window.py — форма добавления/редактирования товара
  orders_window.py       — окно заказов
db/
  schema.sql             — описание структуры БД
  seed.py                — наполнение тестовыми данными
  shoe_store.db          — готовая база
resources/               — иконки, фото товаров
```

## База данных

10 таблиц, нормализованных до 3НФ:

- Справочники: `roles`, `categories`, `manufacturers`, `suppliers`, `units`, `pickup_points`
- Главные: `users`, `products`, `orders`
- Связующая: `order_items` (состав заказа)

Связи реализованы через внешние ключи с правилами `ON DELETE RESTRICT` (для товаров, ссылающихся из заказов) и `ON DELETE CASCADE` (для позиций заказа).
